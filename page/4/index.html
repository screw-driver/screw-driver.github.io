20200218-选择排序

选择排序

思想

假设序列长度为n，每一趟在n-i个数据中通过比较得到最小值的下标，并将其与序列中第i 个值交换。
选择排序的特点：每一趟中，比较操作全部需要进行，但交换操作只需进行一次。

实例

关键字序列：{4，3，1，5，2}：

设最小值的下标为0；

  1. 最小值4和第二位数3比较，4大于3，更改，最小值下标为1；
  2. 最小值3和第三位数1比较，3大于1，更改，最小值下标为2；
  3. 最小值1和第四位数5比较，1不大于5，不更改，最小值下标为2；
  4. 最小值1和第五位数2比较，1不大于2，不更改，最小值下标为2；
  5. 将下标为0的数与下标为2的数交换，得{1，3，4，5，2}；

此时第一位数为最小值。

设最小值的下标为1；

  1. 最小值3和第三位数4比较，3不大于4，不更改，最小值下标为1；
  2. 最小值3和第四位数5比较，3不大于5，不更改，最小值下标为1；
  3. 最小值3和第五位数2比较，3大于2，更改，最小值下标为4；
  4. 将下标为1的数与下标为4的数交换，得{1，2，4，5，3}；

此时第二位数为第二小值。

设最小值的下标为2；

  1. 最小值4和第四位数5比较，4不大于5，不更改，最小值下标为2；
  2. 最小值4和第五位数3比较，4大于3，更改，最小值下标为4；
  3. 将下标为2的数与下标为4的数交换，得{1，2，3，5，4}；

此时第三位数为第三小值。

设最小值的下标为3；

  1. 最小值5和第五位数4比较，5大于4，更改，最小值下标为4；
  2. 将下标为3的数与下标为4的数交换，得{1，2，3，4，5}；

至此，排序完毕。

代码
复杂度
最坏时间复杂度

序列逆序，如{5，4，3，2，1}时，得到最坏时间复杂度。
设序列内数的数量为n：
1 + 1+(n-1)*10 + (n-1)+((n-1)+(n-2)+…+2+1)*4=2n^2-9n-9
即为O(n^2)。
选择排序减少大量交换操作，相对来说，效率比冒泡排序高。

空间复杂度

算法执行所需要的临时空间不随着输入规模的大小而变化，即此算法空间复杂度为一个常量，可表示为 O(1)。

20200218-冒泡排序改进

冒泡排序改进
思想

若数列循环比较一遍，发现没有进行交换操作的，则说明序列已经正序，后续的循环比较操作也无需再进行。

实例

关键字序列：{2，1，3，4，5}：

  1. 第一位数2和第二位数1比较，2大于1，交换，得{1，2，3，4，5}；
  2. 第二位数2和第三位数3比较，2不大于3，不交换，得{1，2，3，4，5}；
  3. 第三位数3和第四位数4比较，3不大于4，不交换，得{1，2，3，4，5}；
  4. 第四位数4和第五位数5比较，4不大于5，不交换，得{1，2，3，4，5}；

第一遍循环比较有进行交换操作，说明序列不一定正序。

  1. 第二位数2和第三位数3比较，2不大于3，不交换，得{1，2，3，4，5}；
  2. 第三位数3和第四位数4比较，3不大于4，不交换，得{1，2，3，4，5}；
  3. 第四位数4和第五位数5比较，4不大于5，不交换，得{1，2，3，4，5}；

第二遍循环比较没有进行交换操作，说明序列已经正序，后续的循环比较无需再进行。

代码
复杂度
最坏时间复杂度

序列逆序，如{5，4，3，2，1}时，得到最坏时间复杂度。
设序列内数的数量为n：
1+(n-1)*4 + (n-1)+((n-1)+(n-2)+…+2+1)*12=6n^2-n-4
即为O(n^2)。

空间复杂度

算法执行所需要的临时空间不随着输入规模的大小而变化，即此算法空间复杂度为一个常量，可表示为 O(1)。

20200218-简单排序

简单排序

思想

让每一个关键字都和它后面的每一个关键字比较，如果大则交换，这样第一位置的关键字在第一次循环后一定变成最小值；第二位置的关键字在第二次循环后变成第二小值；后续同理。

实例

关键字序列：{4，3，1，5，2}：

  1. 第一位数4和第二位数3比较，4大于3，交换，得{3，4，1，5，2}；
  2. 第一位数3和第三位数1比较，3大于1，交换，得{1，4，3，5，2}；
  3. 第一位数1和第四位数5比较，1不大于5，不交换，得{1，4，3，5，2}；
  4. 第一位数1和第五位数2比较，1不大于2，不交换，得{1，4，3，5，2}；

此时，第一位数必然是最小值。

  1. 第二位数4和第三位数3比较，4大于3，交换，得{1，3，4，5，2}；
  2. 第二位数3和第四位数5比较，3不大于5，不交换，得{1，3，4，5，2}；
  3. 第二位数3和第五位数2比较，3大于2，交换，得{1，2，4，5，3}；

此时，第二位数必然是第二小的。

  1. 第三位数4和第四位数5比较，4不大于5，不交换，得{1，2，4，5，3}；
  2. 第三位数4和第五位数3比较，4大于3，交换，得{1，2，3，5，4}；

此时，第三位数必然是第三小的。

  1. 第四位数5和第五位数4比较，5大于4，交换，得{1，2，3，4，5}；

至此，排序完毕。

代码
复杂度
最坏时间复杂度

序列逆序，如{5，4，3，2，1}时，得到最坏时间复杂度。
设序列内数的数量为n：
1+(n-1)*2 + (n-1)+((n-1)+(n-2)+…+2+1)*9=4.5n^2-1.5n-2
即为O(n^2)。

空间复杂度

算法执行所需要的临时空间不随着输入规模的大小而变化，即此算法空间复杂度为一个常量，可表示为 O(1)。

缺陷

在确定最小值过程中的数据交换，仅为得到最小值，而对后续数值（第二小值、第三小值等）的确定没有帮助。
在确定第二小值过程中的数据交换，仅为得到第二小值，而对后续数值（第三小值、第四小值等）的确定没有帮助。
后续同理。

冒泡排序改善上述缺陷。

20200206-冒泡排序

冒泡排序

思想

两两比较相邻记录的关键字，如果反序则交换，直到没有反序的记录为止.

实例

关键字序列：{4，3，1，5，2}：

  1. 第一位数4和第二位数3比较，4大于3，交换，得{3，4，1，5，2}；
  2. 第二位数4和第三位数1比较，4大于1，交换，得{3，1，4，5，2}；
  3. 第三位数4和第四位数5比较，4不大于5，不交换，得{3，1，4，5，2}；
  4. 第四位数5和第五位数2比较，5大于2，交换，得{3，1，4，2，5}；

此时，最后一位数必然是最大值。

  1. 第一位数3和第二位数1比较，3大于1，交换，得{1，3，4，2，5}；
  2. 第二位数3和第三位数4比较，3不大于4，不交换，得{1，3，4，2，5}；
  3. 第三位数4和第四位数2比较，4大于2，交换，得{1，3，2，4，5}；

此时，最后第二位数必然是第二大值。

  1. 第一位数1和第二位数3比较，1不大于3，不交换，得{1，3，2，4，5}；
  2. 第二位数3和第三位数2比较，3大于2，交换，得{1，2，3，4，5}；

此时，最后第三位数必然是第三大值。

  1. 第一位数1和第二位数2比较，1不大于2，不交换，得{1，2，3，4，5}；

至此，排序完毕。

代码
复杂度
最坏时间复杂度

序列逆序，如{5，4，3，2，1}时，得到最坏时间复杂度。
设序列内数的数量为n：
1+(n-1)*2 + (n-1)+((n-1)+(n-2)+…+2+1)*11=5.5n^2-2.5n-2
即为O(n^2)。

空间复杂度

算法执行所需要的临时空间不随着输入规模的大小而变化，即此算法空间复杂度为一个常量，可表示为 O(1)。

缺陷

当序列为{1，2，3，4，5}时，虽然不会进行交换操作，但依旧会进行所有的比较操作，而这些比较操作大部分本是可以省略的。
