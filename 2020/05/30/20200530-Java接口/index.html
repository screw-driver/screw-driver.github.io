Java接口

Java接口是什么？

接口不是类，而是对希望实现某个接口的类的一组需求。

为什么提出Java接口？

有些程序设计语言（如C++）允许一个类有多个直接父类，即多重继承特性。而Java设计者为保持Java语言的简洁性选择不支持多重继承。因此Java规定每个类只能扩展一个类，即每个类只能有一个直接父类。

Java单继承特性限制类定义行为的灵活性，为了能够灵活地为类定义行为，Java设计者提出接口概念并规定一个类可以实现多个接口。

Java接口如何申明？

声明接口的语法格式：

修饰符说明
  • 修饰符可以是public或者省略（即包权限访问控制符）。
接口名说明
  • 由多个有意义的单词连缀而成，每个单词首字母大写。
extends说明
  • extends关键字用于接口继承。
  • 接口支持多继承。
成员变量说明
  • 接口声明里的成员变量仅能是共有静态常量，因此隐式声明为public static final。
  • 因为接口声明里没有构造器和初始化块，因此接口里定义的成员变量只能在定义时指定默认值。
方法说明
  • 接口里的方法仅能是抽象实例方法、私有方法、默认方法、类方法。
方法名 修饰符 实现方法体
抽象实例方法 (public abstract) 不准
私有方法 private [static] 必须
默认方法 (public) default 必须
类方法 `[(public) private] static`

()内的修饰符是隐式声明，[]内的修饰符是可有可无的或多选一。

默认方法的作用

  • 类的实现者仅想实现接口声明中的部分方法时，这就要求接口声明的设计者必须将可能不会被类实现者实现的方法改成默认方法。
  • “接口演化”，向接口声明中加入新方法时，若该方法不是默认方法则程序无法编译成功。
内部类说明
  • 接口里定义的内部类、内部接口、内部枚举都隐式声明为public static。
类如何实现接口？

语法格式

  • 在类中实现接口声明中的方法时必须把方法声明为public。因为接口声明中方法的访问权限默认都是public，而就像子类重写父类方法一样，类在实现接口声明中的方法时，该方法的访问权限只能更大或相等。

  • 一个类只能有一个父类，但可以同时实现多个接口。

  • 当类实现接口的时候，类要实现接口声明中所有的方法。否则类必须声明为抽象类。

  • 默认方法冲突的解决原则

    • 父类与接口冲突

      若一个类继承一个父类并实现一个接口。若父类提供一个具体的方法，而接口声明中也存在方法签名相同的默认方法。此时优先使用父类的方法。

    • 接口与接口冲突

      若一个类实现两个接口。若一个接口声明提供一个默认方法，而另一个接口声明内也存在一个方法签名相同的方法（不管是不是默认方法），类都必须重写该方法以解决二义性问题。

接口变量的作用是什么？
  • Java能定义接口变量，但不能实例化接口。
  • 接口变量只能引用实现该接口的类对象。
标记接口的作用是什么？
  • 若接口不包含任何方法，则称为标记接口。
  • 标记接口的唯一作用是允许在类型查询中使用instanceof。
